### 二叉树

* 二叉排序树(Binary Sort Tree) BST
```
二叉排序树(Binary Sort Tree) BST，也称二叉查找树，亦称二叉搜索树：

对于二叉排序树的任何一个非叶子节点，要求左子树的值比当前节点的值小，右子树的值比当前节点的值大
```

* 平衡二叉树（self-balancing binary search tree）AVL树
```
1. 它是一棵空树，或者左右两棵子树高度差的绝对值不超过1

2. 并且左右两棵子树都是一棵平衡二叉树
```

* B+树
```
1. 所有数据存储在叶子节点的链表中，且链表的数据是有序的

2. 非叶子节点相当于叶子节点的索引（稀疏索引）
```

* 2-3查找树（平衡树）
```
1. 2-节点：
	含有一个键和两条链，左链接指向2-3树中的键都小于该节点，左链接指向2-3树中的键都大于该节点
    
2. 3-节点：
	含有两个键和三条链，左链接指向的2-3树中的键都小于该节点，中间链接指向的2-3树中键都位于该节点两个键之间
    
	右链接指向的2-3树中的键都大于该节点
    
3. 区别：
	2-3树与普通二叉查找树的最大区别是：
    
	普通二叉查找树是自顶向下生长，而2-3树是自底向上生长。
```

* 红黑树
```
红黑树主要是对2-3树进行编码，红黑树的基本思想是用标准的二叉查找树（完全二节点构成）和一些额外信息（替换-3节点）

来表示2-3树。树中的链接分为2种：

	1）红链接：将2个2-节点连接起来构成一个3-结点
  
	2）黑链接：是2-3树的普通链接
  
红黑树的定义：

	红黑树是含有红黑链接并满足下列条件的二叉查找树(BST,二叉排序树)：
  
	1）红链接均为左链接
  
	2）没有任何一个结点同时和两条红链接相连
  
	3）该树是完美的红黑平衡的，即任何空链接（叶子节点到该叶子节点的父节点之间的链接）到根结点的路径上的黑链接数量相同
  
红黑树的平衡化：

	在对红黑树进行一些增删改的操作之后，很可能出现红色的右链接或者两条连续红色的链接，
  
	而这些不满足红黑树的定义，所以需要对其进行旋转进行修复：
  
	1）左旋
  
		当某个结点左子结点为黑色，右子结点为红色，此时需要左旋
    
	2）左旋的过程：
  
		1. 让x的左结点变为h的右结点：h.right = x.left
    
		2. 让h成为x的左子结点： x.left = h
    
		3. 让h的color属性变成x的color属性：x.color = h.color
    
		4. 让h的color属性变为RED：h.color = true
    
	3）右旋
  
	4）右旋的过程：
  
		1. 让x的右子结点成为h的左子结点：h.left = x.right
    
		2. 让h成为x的右子结点：x.right = h
    
		3. 让x的color变成h的color：x.color = h.color
    
		4. 让h的color为RED：h.color = true

	5) 颜色旋转
  
		当一个结点的左子结点和右子结点的color都是red，也就是出现了临时的4-结点，此时，
    
		需要把左子结点和右子结点的颜色变为black，同时让当前节点的颜色变为红色。相当于把临时的
    
		4-结点变成3个2-节点
```
